API endpoints

  • GET /users: Get json of all users
  • POST /users: Submit a raw JSON body containing a valid body (username, password, email, role fields must be filled out)

    IE: {
    "username" : "eddier", >> lowercase, numbers accepted, minlength 5 max 16
    "password" : "EddieReeseIsDaBest", > >> lower + upper case, numbers, min 6, max 16
    "email" : "eddiereese@gmail.com", >> lowercase, numbers accepted, email format
    "role" : "coach" >> lowercase = "coach" or "athlete"
    }

  • JSON PATCH /users/:id : URL must be /users/+ the full user id. Submit a raw JSON body containing an array with objects containing operation type, path, and the value. Works for add and replace.

    IE: [
    {"op": "replace", "path": "/bio/name" , "value": "examplevalue"},
    {"op": "add", "path": "/bio/interests" , "value": ["coaching"]},
    {"op": "replace", "path": "/bio/location" , "value": "Pittsburgh, PA"}
    ]

  • GET /users/:id : URL must be /users/+ the full user id. Returns a single user.
